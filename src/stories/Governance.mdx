<Heading type='1'>Project Governance</Heading>

The CFPB Design System in React (DSR) is a shared responsibility between the CFPB teams that implement and support the component library in their projects. Contributors to the DSR must follow the [CFPB's Open Source Code of Conduct](https://github.com/cfpb/design-system-react/blob/main/CODE_OF_CONDUCT.md) principles and [contribution guidelines](https://github.com/cfpb/design-system-react/blob/main/CONTRIBUTING.md).

The responsibilities of the core contributors can include:
- Attending the DSR's biweekly meetings to discuss updates, blockers, and planning
- Reviewing and merging pull requests
- Verifying new components against the CFPB Design System
- Triaging security issues and bugs
- Updating documentation and helping onboard new users and projects

For a full list of past and current contributors, see the [GitHub repository](https://github.com/cfpb/design-system-react/graphs/contributors).

<Heading type='2'>Meetings</Heading>

There's a biweekly meeting for DSR contributors to discuss updates, blockers, and planning. See the meeting invite for details and a link to the meeting notes. If you are a CFPB employee interested in joining the meeting, contact Bill Himmelsbach or Natalia Fitzgerald.
